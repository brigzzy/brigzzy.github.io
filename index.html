Configuring Bitbucket SSH with Multiple Keys

6 June, 2019
If you use Bitbucket in multiple environments (for example home, and work), you may need to configure multiple SSH keys. This is because once a key has been added to bitbucket (be it a work account, or a personal one), that key can no longer be used in any other Bitbucket account. The problem is that when you try and clone a repo via SSH, you may get a permission denied error when you connect to bitbucket.
